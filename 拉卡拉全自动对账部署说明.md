# 拉卡拉全自动对账部署说明

网站已接入两种拉卡拉对账方式：

1. `申请全日对账文件`：推荐方式，用拉卡拉「申请生成对账文件」接口，适合每天自动对账。
2. `单笔核验已填流水`：备用方式，用拉卡拉「统一查询」接口，适合某一笔成交精确核验。

## 云函数文件

- `supabase/functions/lakala-recon-apply/index.ts`
  - 向拉卡拉申请生成某天的交易对账文件。
  - 调用接口：`https://s2.lakala.com/api/v3/bmmp4/checkFile/apply`

- `supabase/functions/lakala-recon-callback/index.ts`
  - 接收拉卡拉异步回调。
  - 下载 `downloadUrl` 对账文件。
  - 解析流水金额和流水号。
  - 写回 Supabase 的 `sales_dashboard_state` 云端数据。

- `supabase/functions/lakala-verify/index.ts`
  - 单笔流水号核验，备用。

## Supabase Secrets

在 Supabase 项目后台：

`Project Settings -> Edge Functions -> Secrets`

添加：

- `LAKALA_MERCHANT_NO`：拉卡拉商户号
- `LAKALA_TERM_NO`：拉卡拉终端号，没有可先不填
- `LAKALA_APPLY_URL`：默认可不填，默认生产地址是 `https://s2.lakala.com/api/v3/bmmp4/checkFile/apply`
- `LAKALA_QUERY_URL`：默认可不填，默认生产地址是 `https://s2.lakala.com/api/v3/searcher/base_core/trans_query`
- `LAKALA_CALLBACK_TOKEN`：自己设置一个随机字符串，用于保护回调接口
- `LAKALA_CALLBACK_URL`：回调函数公开地址，例如：

```text
https://你的项目ID.supabase.co/functions/v1/lakala-recon-callback
```

- `LAKALA_HEADERS_JSON`：如果拉卡拉要求额外鉴权头，就放这里，例如：

```json
{
  "Authorization": "Bearer xxxxx"
}
```

> 注意：商户密钥、私钥、证书不要放进 `index.html`，也不要上传到 GitHub。

## 部署命令

在本文件夹下执行：

```bash
supabase login
supabase link --project-ref 你的Supabase项目ID
supabase functions deploy lakala-recon-apply
supabase functions deploy lakala-recon-callback --no-verify-jwt
supabase functions deploy lakala-verify
```

`lakala-recon-callback` 必须允许拉卡拉服务器访问，所以要用 `--no-verify-jwt`，但我们用 `LAKALA_CALLBACK_TOKEN` 做保护。

## 网站里怎么用

1. 打开网站。
2. 点 `💳 拉卡拉对账`。
3. 选择日期。
4. 点 `申请全日对账文件`。
5. 拉卡拉生成完成后，会回调 `lakala-recon-callback`。
6. 回调函数会把流水自动写进 Supabase。
7. 网站实时同步后会显示金额是否对平。

## 每天晚上全自动

要做到每天晚上自动跑，需要让 Supabase 定时调用：

`lakala-recon-apply`

建议时间：

- 每天 23:55 申请当天文件
- 或第二天 01:00 申请前一天文件

如果拉卡拉文件生成有延迟，建议用第二天 01:00。

## 目前还需要确认

拉卡拉这个文档只说明了申请文件和回调下载地址，没有说明下载文件内部字段格式。

我已经做了通用解析：

- 支持制表符、逗号、竖线分隔
- 自动识别：交易流水号、对账单流水号、交易金额、实收金额、支付状态、交易时间

如果正式下载下来的文件格式不同，需要拿一份样例文件再微调解析规则。
